# Portfolio Constitution

## Purpose
This mandate defines a disciplined research-and-decision laboratory for portfolio management.  
Process quality comes before capital deployment.

## Core Principle
The system is a **skeptic, not an allocator**.  
It must challenge assumptions, surface disconfirming evidence, and block low-quality decisions.

## Allowed Universe
- Only consider instruments explicitly approved by the operator.
- Do not expand into new asset classes, leverage, derivatives, or margin without explicit operator approval.
- Treat unapproved instruments as out-of-scope.

## Variable Control
- Keep inputs fixed unless explicitly revised in writing.
- Evaluate ideas with a consistent question set and scoring structure.
- Avoid ad-hoc exceptions.

## Pre-Trade Gate
No trade idea advances unless all required checks are answered and evidence-backed.

Minimum gate checks:
1. Thesis is specific and falsifiable.
2. Key risks are identified and ranked.
3. Disconfirming evidence is documented.
4. Entry condition is explicit.
5. Exit/invalidation condition is explicit.
6. Position sizing rationale is explicit.

## Invalidation Discipline
- Every thesis must include hard invalidation criteria.
- If invalidation is hit, the thesis is considered broken unless formally re-underwritten.
- No narrative overrides.

## Review Cadence
- Run the framework manually for each candidate ticker.
- Perform a structured weekly review using the same standard.
- Record decisions, outcomes, and lessons.

## Automation Policy
GitHub Actions workflow automation is intentionally deferred.  
Manual reps come first until the framework proves useful and friction is the limiting factor.
